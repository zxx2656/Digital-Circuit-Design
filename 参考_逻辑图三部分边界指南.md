# 逻辑图三部分边界指南

> 每部分画到什么程度、该体现什么、不该体现什么

---

## 一句话判断标准

```
如果你在画第一部分时画了电阻 → 错了，那是第二部分的事
如果你在画第二部分时画了 PCB 布局 → 错了，那是以后画 PCB 时的事
如果你在画第三部分时画了引脚号 → 错了，那是引脚约束表的事
```

---

## 第一部分：核心板+底板的系统输入输出

### 画成什么样

**一个黑色大方块（整个电路板），左边伸进几根箭头（输入），右边伸出几根箭头（输出）。**

```
        输入                    输出
         │         ┌─────────┐     │
  Type-C线 ───────▶│         │────▶ 数码管
         │         │ 整个板子 │     │
  开关1   ───────▶│ (黑盒子) │────▶ 绿灯
         │         │         │     │
  开关2   ───────▶│         │────▶ 红灯
         │         └─────────┘     │
```

### 要体现的内容

| 要体现 | 具体到什么程度 |
|--------|--------------|
| 输入有哪些 | Type-C 线（5V供电+JTAG下载）、两个拨动开关 |
| 输出有哪些 | 4位数码管、绿色LED、红色LED |
| 每个输入/输出的**作用** | 开关1=清零和启动、开关2=暂停和恢复；绿灯=计数中，红灯=暂停中，都灭=待机 |
| 核心板的位置 | 标出"核心板(90×60)架在底板(80×80)上方"即可，不标位置坐标 |
| 设计理由（文字） | 为什么用拨动开关、为什么用红绿灯、为什么不用外部晶振 |

### 不要体现的内容

| 不要体现 | 为什么 |
|----------|--------|
| 电阻值、三极管 | 那是电路细节，第二部分的事 |
| 数码管具体放在底板哪个位置 | 那是 PCB 布局的事 |
| FPGA 内部有什么模块 | 那是第三部分的事 |
| 排针排母的数量 | 那是第二部分的事 |
| 信号名（SEG[7:0] 等） | 那是第二部分的事 |

### 精细度的比喻

**相当于你在向一个不懂电路的人介绍这个产品：**
- "这个秒表是这样用的：插上线，拨这个开关开始计时，数码管显示时间"
- "绿灯亮说明在走，红灯亮说明停了，都不亮说明还没开始"

## 第二部分：底板的输入输出

### 画成什么样

**核心板排母信号列表 + 底板器件连接框图。** 能看到"什么信号从哪个 FPGA 引脚出来，经过什么器件，到达哪个底板器件"。

```
核心板(FPGA引脚)         底板器件
   │                      │
   SEG[7:0] ──── 470Ω×8 ────▶ 数码管段选(a~g,dp)
   DIG[3:0] ──── 4.7KΩ ──── S8550 ────▶ 数码管位选(4位)
   LED_RUN  ──── 470Ω ────▶ 绿色LED → GND
   LED_PAUSE─── 470Ω ────▶ 红色LED → GND
   SW_RST ◀──── 10KΩ上拉3.3V ──── 拨动开关1 → GND
   SW_PAUSE◀──── 10KΩ上拉3.3V ──── 拨动开关2 → GND
```

### 要体现的内容

| 要体现 | 具体到什么程度 |
|--------|--------------|
| **每个信号的名称** | SEG[7:0]、DIG[3:0]、LED_RUN、LED_PAUSE、SW_RST、SW_PAUSE |
| **每个信号的方向** | FPGA→底板（输出）、底板→FPGA（输入） |
| **每个信号的位宽** | SEG 是 8bit、DIG 是 4bit、其余是 1bit |
| **经过什么器件** | 串 470Ω 电阻、经 4.7KΩ 到 S8550 基极、上拉 10KΩ 到 3.3V |
| **器件的参数值** | 电阻=470Ω/4.7KΩ/10KΩ、三极管=S8550(PNP) |
| **最终连到哪个实物** | 数码管第 a 段、数码管位1(分十位)、绿色LED 正极 |
| **电源从哪里来** | 5V 和 3.3V 从核心板排针引出 |
| **设计理由（文字）** | 为什么段选要 470Ω、为什么位选要 S8550、为什么开关要上拉 |

### 不要体现的内容

| 不要体现 | 为什么 |
|----------|--------|
| 数码管放在底板左边还是右边 | 那是 PCB 布局的事 |
| 具体引脚号（IO15、IO25...） | 那是引脚分配表的事，原理图阶段才填 |
| FPGA 内部模块怎么连 | 那是第三部分的事 |
| 底板尺寸、固定孔位置 | 那是 PCB 机械层的事 |
| PCB 走线怎么走 | 那是画 PCB 的事 |

### 精细度的比喻

**相当于你在告诉一个画原理图的人：**
- "SEG_a 这条线从 FPGA 出来，串一个 470Ω 电阻，接到数码管的 a 段引脚"
- "DIG[0] 从 FPGA 出来，经过 4.7KΩ，接到 S8550 的基极，S8550 的集电极接 5V，发射极接到数码管的第一位阳极"
- "SW_RST 这条线，开关闭合时接地(低电平)，断开时通过 10KΩ 拉到 3.3V(高电平)，然后进 FPGA"

## 第三部分：核心板上程序的逻辑图

### 画成什么样

**FPGA 内部的 5 个功能方块，方块间有信号箭头连接。** 每个方块标注模块名，每根箭头标注信号名。

```
       50MHz                     SW_RST  SW_PAUSE
         │                          │      │
    ┌────▼────┐                ┌────▼──────▼────┐
    │ clk_div │                │ key_debounce   │
    │秒脉冲发生器│               │   按键消抖      │
    └────┬────┘                └────┬──────┬────┘
         │ clk_1Hz           sw_rst_clean  sw_pause_clean
         │                          │      │
         │                     ┌────▼──────▼────┐
         │                     │   fsm_ctrl     │
         │                     │   控制状态机     │
         │                     └──┬────┬────┬───┘
         │              count_en │    │    │ led_run(绿)
         │           count_clr ──┘    │    │ led_pause(红)
         │                            │    │
    ┌────▼────────────────────────────▼────┐
    │           counter_60                 │
    │          60进制BCD计数器              │
    └────────────────┬─────────────────────┘
                     │ BCD[15:0]
    ┌────────────────▼─────────────────────┐
    │           seg_decoder                │
    │           显示译码器                   │
    └────────┬────────────────┬────────────┘
             │ SEG[7:0]       │ DIG[3:0]
```

### 要体现的内容

| 要体现 | 具体到什么程度 |
|--------|--------------|
| 有几个模块 | 5 个：clk_div、key_debounce、fsm_ctrl、counter_60、seg_decoder |
| 每个模块的**功能名称** | "秒脉冲发生器"、"按键消抖"、"控制状态机"、"60进制BCD计数器"、"显示译码器" |
| 每个模块的**输入信号** | clk_div：clk_50MHz；counter_60：clk_1Hz, count_en, count_clr |
| 每个模块的**输出信号** | clk_div：clk_1Hz；fsm_ctrl：count_en, count_clr, led_run, led_pause |
| 信号之间的**连接关系** | clk_div 的 clk_1Hz 同时给 counter_60、seg_decoder |
| 状态机的**状态转移图** | IDLE → COUNT → PAUSE，三个状态之间的跳转条件 |
| 信号名标注 | 必须标在箭头旁边 |
| 输出到 FPGA 外部 | SEG[7:0]、DIG[3:0]、LED_RUN、LED_PAUSE 标为最终输出到引脚 |

### 不要体现的内容

| 不要体现 | 为什么 |
|----------|--------|
| 每个模块的内部 Verilog 代码 | 那是下一步写代码的事。逻辑图只要方块+信号，不要代码 |
| 引脚号（IO15） | 那是引脚分配表的事 |
| 电阻值、三极管 | 那是第二部分的事 |
| 数码管实物位置 | 那是第一/二部分和 PCB 的事 |
| FPGA 外部连接 | 那是第二部分的事 |

### 精细度的比喻

**相当于你在向一个要写 Verilog 的人交代任务：**
- "你要写 5 个 .v 文件，名字分别是 clk_div, key_debounce, fsm_ctrl, counter_60, seg_decoder"
- "clk_div 的输入是 clk_50MHz，输出是 clk_1Hz"
- "fsm_ctrl 有一个状态机，三个状态 IDLE/COUNT/PAUSE，输出 count_en 给 counter_60"
- "counter_60 输出 16bit 的 BCD 码，格式是 {min_h, min_l, sec_h, sec_l}"

## 三部分边界对比一览

```
第一部分（最宏观）
  画什么：  ● Type-C线、开关 = 输入
           ● 数码管、LED = 输出
           ● 核心板在底板上方

  不画：    ✗ 电阻值、三极管
           ✗ 信号名
           ✗ FPGA内部模块
           ✗ 器件位置

第二部分（中间层）
  画什么：  ● 每个信号名 + 位宽 + 方向
           ● 经过的器件（电阻/三极管）
           ● 电阻值（470Ω/4.7KΩ/10KΩ）
           ● 电源来向（从核心板排针取）

  不画：    ✗ 器件物理位置
           ✗ 引脚号
           ✗ FPGA内部模块
           ✗ Verilog代码

第三部分（最微观，仅限FPGA内部）
  画什么：  ● 5个功能方块
           ● 每个方块的输入输出信号名
           ● 信号之间的连接
           ● 状态转移图

  不画：    ✗ 引脚号
           ✗ 电阻值
           ✗ 硬件器件
           ✗ Verilog代码
```

## 回答你的具体问题

| 问题 | 答案 |
|------|------|
| 是否画核心板接在底板上？ | **第一部分**：画，标"核心板通过排针排母架在底板上方"。**第二部分**：画排母接口的信号表。**第三部分**：不画 |
| 要说清数码管放哪里吗？ | 第一部分：说"数码管作为输出显示 mm:ss"即可。**不说**放左边还是右边。"放哪里"是画 PCB 时的事 |
| 开关的作用要写吗？ | 第一部分：写，"开关1=清零/启动，开关2=暂停/恢复"。第二部分：写，"SW_RST 上拉到 3.3V，闭合=低电平=清零" |
| 要连接不同器件吗？ | 第二部分：**要**，画出"FPGA→470Ω→数码管段选"这样的信号路径。第一部分不要，第三部分不要 |
| 精细到什么程度？ | 第一部分≈"产品说明书"，第二部分≈"给画原理图的人的工作指令"，第三部分≈"给写 Verilog 的人的任务书" |

---

> 简单记：第一部分给老师看你的设计思路，第二部分给画 AD 的人（你自己）看怎么连线，第三部分给写代码的人（你自己）看要写几个模块。
