# 课设完整流水线

> 从零开始到验收通过，每一步干什么、产出什么、对后面有什么用

---

## 总览

```
第一步：逻辑图 ──▶ 第二步：AD原理图+Verilog ──▶ 第三步：制板焊接 ──▶ 第四步：下载调试验收
  (现在要做)        (并行工作)                    (工厂+手工)          (最终验证)
```

## 第一步：逻辑图（你现在要做的事）

**不需要打开任何软件，纸+笔 或 Draw.io 就够。**

### 三部分内容

| 部分 | 画什么 | 回答什么问题 | 对后面的作用 |
|------|--------|-------------|-------------|
| **1. 核心板+底板的输入输出** | 整个板子对外：左边什么进去，右边什么出来 | 底板上要焊哪些器件？ | 决定器件清单，原理图里知道要放什么 |
| **2. 底板的输入输出** | 核心板排母 ↔ 底板器件之间的信号和电源 | 每根信号线接哪个器件？电阻多大？三极管怎么接？ | **直接指导 AD 原理图怎么画** |
| **3. 核心板上程序的逻辑图** | FPGA 内部 5 个功能模块 + 信号流向 | 要写几个 .v 文件？每个模块的输入输出是什么？ | **直接指导 Verilog 代码怎么写** |

### 还要搞清的事

| 序号 | 要弄清 | 去哪里找 |
|------|--------|---------|
| ① | 核心板排母有哪些可用 GPIO？ | 核心板引脚定义文档 |
| ② | GPIO 电平标准？(3.3V？) | 核心板手册 |
| ③ | 共需要多少 I/O？ | 功能性数：2开关 + 8段选 + 4位选 + 3LED = 17个 |
| ④ | 开关上拉还是下拉？ | 查器件手册 + 电路常识 |
| ⑤ | 共阳极数码管段选低有效还是高有效？ | 查 SR410361N 数据手册 |

### 产出

```
一份文档，包含三张图 + 设计理由说明
→ 就是 逻辑图设计_三层结构.md 那个文件
```

## 第二步：并行工作

**逻辑图确定之后，两条线同时走，互不依赖。**

### 2A：Altium Designer — 画原理图 + PCB

```
输入：逻辑图第2部分（底板信号连接图） + 核心板引脚表

原理图做的事：
  ┌────────────────────────────────────────────┐
  │  从 SchLib 库拖器件符号                     │
  │  按逻辑图的信号表连线                        │
  │  标注网络标签（信号名）                       │
  │  标注电阻值（470Ω、4.7KΩ、10KΩ）             │
  │  标注引脚号（等拿到核心板引脚表后补）           │
  └────────────────────────────────────────────┘

  ↓ Design → Update PCB Document

PCB 做的事：
  ┌────────────────────────────────────────────┐
  │  器件布局（数码管放哪、开关放哪、LED放哪）     │
  │  设定线宽和间距规则                          │
  │  布线（双面板，手动走线）                     │
  │  覆铜（铺地）                                │
  │  DRC 检查                                   │
  └────────────────────────────────────────────┘

产出：.SchDoc（原理图） + .PcbDoc（PCB）
```

### 2B：高云 EDA — 写 Verilog 代码

```
输入：逻辑图第3部分（FPGA 内部模块图） + 核心板引脚表

做的事：
  ┌────────────────────────────────────────────┐
  │  照着逻辑图里的 5 个方块，建 5 个 .v 文件      │
  │  clk_div.v    — 秒脉冲发生器                │
  │  key_debounce.v — 按键消抖                  │
  │  fsm_ctrl.v   — 控制状态机                  │
  │  counter_60.v — 60进制BCD计数器             │
  │  seg_decoder.v — 显示译码+动态扫描           │
  │  top.v        — 顶层，把上面 5 个连起来      │
  │                                            │
  │  写 Testbench 仿真验证每个模块               │
  │  综合 → 布局布线 → 引脚约束（按引脚表分配）    │
  │  生成比特流文件                              │
  └────────────────────────────────────────────┘

产出：.v 源文件 + 比特流文件（.fs）
```

### 两条线在这里交汇

```
AD 原理图里的引脚号   =   高云 EDA 里 pin planner 的引脚约束

         ↕ 必须一致！否则烧进去对不上
```

## 第三步：制板 + 焊接

```
AD 导出 Gerber 文件 → 发给 PCB 工厂 → 工厂做出裸板（等 3~7 天）

拿到裸板后：
  ┌────────────────────────────────────────────┐
  │  焊接器件（按原理图和元件清单）：              │
  │  排母 × 2                                   │
  │  数码管 × 1                                 │
  │  拨动开关 × 2                               │
  │  LED × 3（红×2 + 绿×1）                     │
  │  电阻 470Ω、4.7KΩ、10KΩ                      │
  │  三极管 S8550 × 4                           │
  │                                            │
  │  注意：先焊矮的（电阻），再焊高的（排母、数码管）│
  │       焊完先不插核心板，万用表测电源和地是否短路 │
  └────────────────────────────────────────────┘

此时：硬件已存在，但 FPGA 是空的，什么都不会做
```

## 第四步：下载 + 调试 + 验收

```
插入核心板 → USB 线连电脑 → 高云 EDA 下载比特流到 FPGA

逐项测试 5 个功能：

  ┌────┬──────────┬─────────────────────────┐
  │ #  │   功能    │        怎么测             │
  ├────┼──────────┼─────────────────────────┤
  │ 1  │ 计时范围  │ 启动后看能不能从 00:00    │
  │    │ 59:59→00 │ 走到 59:59 再回到 00:00   │
  ├────┼──────────┼─────────────────────────┤
  │ 2  │ 任意清零  │ 计到一半拨开关1，看是否    │
  │    │          │ 立刻 00:00               │
  ├────┼──────────┼─────────────────────────┤
  │ 3  │ 暂停恢复  │ 计时中拨开关2，看是否停住   │
  │    │          │ 再拨回去，看是否继续        │
  ├────┼──────────┼─────────────────────────┤
  │ 4  │ 预置值    │ 改 INIT 参数重下载，启动后 │
  │    │          │ 看是否从预设值开始计        │
  ├────┼──────────┼─────────────────────────┤
  │ 5  │ LED 指示  │ 秒脉冲灯是否闪、运行灯是否亮 │
  │    │          │ 暂停灯切换是否正常          │
  └────┴──────────┴─────────────────────────┘

不对就改 Verilog → 重新下载 → 再测 → 直到全部正常

全部通过 → 写设计报告 → 验收
```
