# 秒表（Stopwatch）逻辑图设计

> 基于高云 GW1N-UV9EQ144C6/I5 FPGA 核心板
> 数字逻辑课程设计

---

## 一、设计任务和要求

以高云 GW1N-UV9EQ144C6/I5 为核心板，用中、小规模集成电路设计分秒计数时钟，能实现以下功能：

| 序号 | 功能 | 逻辑设计对应 |
|------|------|-------------|
| 1 | 能计时1小时，即 59:59 时返回 00:00，并用 LED 数码管实时显示计数值 | counter_60 模块：60进制计数器 + seg_decoder 模块：实时译码显示 |
| 2 | 计数初值通常从 00:00 开始，并能在任意时刻任意计数值时返回初值 00:00 | fsm_ctrl 模块：开关1控制清零，任意状态→IDLE→count_clr=1→归零 |
| 3 | 能在任意时刻暂停计数并显示计数值，又能随时恢复计数 | fsm_ctrl 模块：开关2控制暂停/恢复，COUNT↔PAUSE |
| 4 | 可以预置分秒的任意时间值，如 15:45 | 设计方案提示指定：通过重新下载程序给时钟设置初始值（改 Verilog 参数 → 重新下载）。如需硬件预置可扩展 |
| 5 | 可以设置适当的发光二极管用来指示工作状态 | LED_1HZ(秒脉冲闪烁) + LED_RUN(运行常亮) + LED_PAUSE(暂停常亮) |

## 二、设计成果

| 序号 | 交付物 | 说明 |
|------|--------|------|
| 1 | 秒表电路板 | 底板设计+焊接，开学后调试通过验收 |

## 三、设计方案提示（原始要求）

```
① 产生秒脉冲发生器，该秒脉冲作为计数器的时钟脉冲。
② 设置两个开关，一个用于清零和启动计数；另一个用于暂停和恢复计数。
   另外可以通过重新下载程序给时钟设置初始值。
③ 启动计数后，计数器进行加1计数，当由59s进到60s时，自动清零循环计数，分钟也是如此。
④ 计数器的实时计数状态经程序设计显示译码器进行译码，再送到LED数码管显示出来。
```

**从这4条中提取的系统信号链：**

```
50MHz晶振 → [秒脉冲发生器] → 1Hz → [计数器 00:00~59:59] → BCD码 → [显示译码器] → 段选/位选 → [LED数码管]
                                    ↑
                              开关1 (清零/启动)
                              开关2 (暂停/恢复)
                                    ↓
                              [控制状态机]
```

| 参数 | 对底板/逻辑设计的影响 |

## 第一层：核心板 + 底板 — 系统输入与输出

本层将整个秒表电路板视为黑盒子，只关心**外部向板子输入什么，板子向外输出什么**。

> **关键理解**：核心板自带 50MHz 晶振、自带 Flash、自带电源接口（Type-C）。
> 所以"外部输入"不是晶振、不是 5V 电源芯片——这些都在核心板上了。
> 真正的"外部输入"只有一根 Type-C 线 + 人的手指。

### 为什么这样设计

| 设计决策 | 理由 |
|----------|------|
| 外部输入只有一根 Type-C 线 | 核心板自带 50MHz 晶振、自带电源管理、自带 Flash——底板不需要额外供电口或晶振。Type-C 既是电源又是下载口，一根线解决所有 |
| 输入用拨动开关而非按键 | 清零和暂停需要**保持状态**，拨动开关有明确的 ON/OFF 位置，用户一眼就知道当前状态 |
| 显示用 4 位数码管 | 时间格式 mm:ss 刚好 4 位数字，共阳极数码管驱动简单、亮度高 |
| LED 三色区分 | 秒脉冲闪灯 = 证实时钟在跑，红灯常亮 = 正在计时，绿灯常亮 = 暂停中，用户扫一眼就知道秒表状态 |
| 预置值靠重新下载程序 | 核心板 Flash 存储程序，改参数重新下载即可。不增加底板硬件复杂度 |
| 核心板(90×60) 大于 底板(80×80) | 核心板**架在底板上方**，通过排针排母连接。底板不需要给核心板留嵌入空间 |

## 第二层：底板 — 输入与输出信号

本层将 FPGA 核心板视为信号源/信号接收方，底板是中间层。**核心板给底板什么？底板给核心板什么？**

> **关键理解**：底板自己不产生任何电压。5V 和 3.3V 都从核心板的排针引过来。

### 为什么底板这样设计

| 设计决策 | 理由 |
|----------|------|
| 底板电源从核心板排针取 | 核心板自带电源管理(Type-C→5V→3.3V)，底板不需要独立的电源芯片或电池。直接从排母的 5V 和 3.3V 引脚取电 |
| 数码管段选各串 470Ω | 共阳极数码管，FPGA I/O 输出低电平点亮。I/O 电压 3.3V，LED 段压降≈2V，电流=(3.3-2)/470≈2.8mA，在 FPGA I/O 安全驱动范围内 |
| 位选经 S8550 三极管放大 | 每位要同时驱动 8 段 LED，8×3mA=24mA，远超 FPGA I/O 上限(8~16mA)。S8550 是 PNP 管，集电极电流可达 500mA，安全余量大 |
| 三极管基极串 4.7KΩ | FPGA 输出低时 B 极≈0V，基极电流=(5-0.7)/4.7K≈0.9mA，放大倍数≈100 → 集电极≈90mA，足够点亮 8 段 LED |
| 开关上拉 10KΩ 到 3.3V | 3.3V 从核心板排针取。10KΩ 保证断开时电平确定(高)，闭合时电流 3.3/10K=0.33mA 功耗极低 |
| LED 各串 470Ω | LED 压降≈2V，电流=(3.3-2)/470≈2.8mA，亮度适中不刺眼 |
| 仅用两个拨动开关 | 设计提示明确只需要清零/启动 和 暂停/恢复。预置值靠重新下载程序到 Flash |
| 底板 ≤ 80×80mm | 核心板 90×60 架在上方，排母位置需对准核心板的排针位置 |

## 第三层：核心板 FPGA 内部 — 程序逻辑图

本层是逻辑设计的**核心**。描述烧录到 FPGA 内的 Verilog 程序由哪些模块组成，模块之间如何连接。

本设计共 **5 个功能模块**，信号流向严格遵循设计提示的"秒脉冲发生器→计数器→显示译码器"主线。

## 和设计提示的逐条对应关系

```
设计提示①：秒脉冲发生器
  → FPGA 内部模块 clk_div，50MHz 分频得到 1Hz，作为 counter_60 的时钟

设计提示②：两个开关 + 重下载设初值
  → 开关1(sw_rst)：   拨到"清零"→fsm_ctrl进入IDLE，count_clr=1，显示00:00
                      拨到"启动"→fsm_ctrl进入COUNT，count_en=1，开始计时
  → 开关2(sw_pause)： 拨到"暂停"→fsm_ctrl进入PAUSE，count_en=0，保持显示
                      拨到"恢复"→fsm_ctrl回到COUNT，继续计时
  → 重下载设初值：    修改 counter_60 模块的初始值参数，重新综合/下载，无需硬件改动

设计提示③：加1计数，59→60自动清零循环
  → counter_60 模块：秒个位 0→9，进位触发秒十位+1；秒十位 0→5，进位触发分+1
  → 特殊边界：当下一个 1Hz 脉冲到来且当前为 59:59 时，直接强制归零到 00:00

设计提示④：译码→LED数码管显示
  → seg_decoder 模块：将 4 位 BCD 码经 7 段译码表转换为段选信号
  → 动态扫描轮流选通 4 个数码管位，频率 >200Hz，人眼看不出闪烁
```
